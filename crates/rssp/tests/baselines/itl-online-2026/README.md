# ITL Online 2026 — StepParity note-annotation baselines

Golden per-row StepParity annotations for the **ITL Online 2026** pack, consumed
by [`crates/rssp/tests/note_annotations_parity.rs`](../../note_annotations_parity.rs)
to verify rssp's per-row annotations against ITGmania.

This is the row-level companion to the aggregate `tech_counts` parity: it pins
down, for every judged row, *which* columns the feet land on, *which foot* lands
on each (inner pivot vs. outer crossed-over arrow), and the *full per-row tech*
breakdown — fidelity that equal aggregate counts alone cannot prove.

## Layout

```
baseline/<md5[0:2]>/<md5>.json.zst
```

- `<md5>` = MD5 of the **raw** simfile bytes (`.ssc`/`.sm`), lowercase hex — the
  same keying rssp's parity test uses to look a baseline up.
- Each file is a zstd-compressed JSON **array** of chart records. Every record
  carries, alongside the other harness metrics:
  ```json
  "note_annotations": [
    { "beat": 4.0, "columns": [0], "feet": [1], "note_count": 1,
      "tech_counts": { "crossovers": 0, "half_crossovers": 0, "full_crossovers": 0,
        "footswitches": 0, "up_footswitches": 0, "down_footswitches": 0,
        "sideswitches": 0, "jacks": 0, "brackets": 0, "doublesteps": 0 } },
    { "beat": 4.5, "columns": [3], "feet": [3], "note_count": 1,
      "tech_counts": { "crossovers": 1, "half_crossovers": 1, "full_crossovers": 0,
        "footswitches": 0, "up_footswitches": 0, "down_footswitches": 0,
        "sideswitches": 0, "jacks": 0, "brackets": 0, "doublesteps": 0 } }
  ]
  ```
  `columns` are 0-indexed and sorted; `feet[i]` is the foot id on `columns[i]`
  (`None=0, LeftHeel=1, LeftToe=2, RightHeel=3, RightToe=4`); `note_count` is the
  number of feet placed on the row; `tech_counts` is the **full** per-row tech
  classification (per-row counts sum to the chart's aggregate `TechCounts`).
  As in the engine, there is no stored crossover flag — derive it from
  `tech_counts.crossovers > 0`.
- 310 simfiles → 310 baseline files (~0.8 MB).

## Source of truth

Generated by the **itgmania-reference-harness**, which compiles ITGmania's own
C++ StepParity foot-placement + tech classifier and emits per-chart JSON. The
per-row data comes from the engine's `Steps:GetNoteAnnotations()` — the same call
Simply&nbsp;Love's `CrossoverCues.lua` reads — so the engine is the authority and
rssp's `step_parity.rs` is the port being checked.

| Component          | Version |
| ------------------ | ------- |
| Song pack          | ITL Online 2026 (310 simfiles) |
| ITGmania engine    | `4ef50bd3` + the `Steps:GetNoteAnnotations()` binding (this feature) |
| Reference harness  | `pnn64/itgmania-reference-harness@5e846ad` + `note_annotations` JSON emission (incl. `feet` + `tech_counts`) |

> The `GetNoteAnnotations()` engine binding and the harness `note_annotations`
> field were added to expose per-row foot placement + full tech and are not yet
> upstreamed. Record the exact engine/harness builds whenever these baselines are
> regenerated — StepParity cost weights drive foot placement, which drives both
> the foot identities and the tech classification.

## Run the parity check

The ITL Online 2026 **song files are not committed** (size + licensing); point
the packs env var at a local copy of the pack. md5 keying ties each baseline to
the exact simfile bytes, so a chart whose bytes don't match simply skips.

```powershell
$env:RSSP_PARITY_PACKS_DIR    = "D:\path\to\songs\ITL Online 2026"
$env:RSSP_PARITY_BASELINE_DIR = "$PWD\crates\rssp\tests\baselines\itl-online-2026\baseline"
cargo test -p rssp --test note_annotations_parity
# -> 310 passed; 0 failed
```

## Regenerate

Mirror the `pnn64/rssp-tests` `gen-baseline.sh` flow against the patched engine:
for each simfile, run the harness, take the MD5 of the raw bytes, and zstd the
JSON to `baseline/<md5[0:2]>/<md5>.json.zst`.
